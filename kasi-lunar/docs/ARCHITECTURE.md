# 아키텍처

## 목적

`kasi-lunar`는 KASI 지원 범위 전체에 걸쳐 음력/양력 변환을 제공하는 TypeScript npm 라이브러리입니다.

구현은 두 가지 경로로 나뒥니다:

- `1000..2050`: `korean-lunar-calendar`에 위임
- `-59..999`: 생성된 KASI 월별 달력 데이터 사용

## 주요 모듈

### 공개 진입점

- `src/index.ts`

패키지 진입점과 공개 타입을 익스포트합니다.

### 변환 엔진

- `src/kasi-lunar-calendar.ts`

메인 런타임 클래스입니다.

주요 역할:

- 지원되는 양력/음력 범위 검증
- 요청을 상위 위임 경로 또는 초기 데이터 경로로 라우팅
- 초기 연도에 대해 양력 날짜를 내부 JDN 기반 조회로 변환
- 초기 음력 날짜를 양력 날짜로 역변환
- 간지 헬퍼 노입

### 초기 KASI 데이터

- `src/early-data.ts`

음력 연도 `-59..999`에 대해 생성된 정적 데이터입니다.

각 연도는 다음을 저장합니다:

- 음력 연도 시작 JDN
- 누적 일수 및 월 서수
- 정렬된 월 항목 목록

각 월 항목은 다음을 저장합니다:

- 숫자형 월
- 윤달 플래그
- 음력 연도 시작으로부터의 오프셋
- 일수 스팬
- KASI 원본 월 레이블 (예: `01`, `正`, `臘`)

### 초기 데이터 생성기

- `scripts/generate-early-data.mjs`

KASI 월별 달력 페이지를 크롤링하여 `src/early-data.ts`를 생성합니다.

주요 역할:

- KASI 월별 페이지 크롤링
- 양력, 음력, 율리우스력 날짜 컨럼 파싱
- 역사적 월 레이블 정규화
- 각 연도별 정렬된 음력 월 재구성
- TypeScript 데이터 파일 출력

## 범위 전략

### 위임 범위

`solarYear >= 1000` 또는 `lunarYear >= 1000`인 경우 `korean-lunar-calendar`를 직접 사용합니다.

현대 범위를 단순하게 유지하고 상위 패키지와 일치시킵니다.

### 초기 범위

`-59..999`는 상위 패키지가 KASI 전체 범위를 커버하지 않으므로 생성된 데이터를 사용합니다.

초기 경로는 다음을 사용합니다:

- 양력 입력에 대한 그레고리우스력 → JDN 변환
- 연도 데이터 이진 탐색
- 매칭된 음력 연도 내 월 오프셋 조회

## 역사적 중의성

일부 고대 KASI 연도는 같은 음력 연도 안에 중복된 숫자 월 식별자를 포함합니다.

예시:

- `正`과 `01`
- `臘`과 `12`

이 때문에 숫자 `year/month/day`만으로는 초기 음력 날짜를 항상 특정할 수 없습니다.

런타임은 `CalendarData`에 `monthLabel`을 보존하며, `setLunarDate(...)`는 중의적인 초기 연도에 대해 선택적 `monthLabel` 인자를 받습니다.

## 검증

주요 명령어:

- `npm run generate:data`
- `npm run check`
- `npm run harness:smoke`
- `npm run harness:report`
- `npm test`

`scripts/verify-build.mjs`는 경계 케이스와 중의적인 초기 월 케이스를 포함합니다.

# 프로젝트 규칙

## 수정 범위

- 명시적으로 요청받지 않는 한 `kasi-lunar/` 내부만 수정
- 참조 폴더는 아이디어 참고용으로만 읽고 수정하지 않음

## 호환성

- `kasi-lunar`를 npm 라이브러리로 사용 가능하게 유지
- 명시적으로 변경이 요청되지 않는 한 `package.json` 진입점과 타입 익스포트 보존
- 파괴적인 리팩터링보다 추가적인 하네스 변경 선호

## 검증 순서

1. `bun run harness:smoke`
2. `bun run harness:report`
3. 동작 또는 공개 API 변경 시 `bun test`

## 문서화

- 작업 루프가 변경되면 하네스 문서 업데이트
- Codex용 문서는 짧고 안정적이며 작업 중심으로 유지
- 같은 안내를 여러 곳에 중복 작성하지 않음

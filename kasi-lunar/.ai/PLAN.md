# 현재 계획

## 목표

- `kasi-lunar`를 TypeScript npm 라이브러리로 배포 가능한 상태 유지
- 지원 범위 전체에 걸쳐 KASI 변환 커버리지 보존
- Codex 하네스를 안정적이고 가볍게 유지

## 단기 작업

- `scripts/generate-early-data.mjs`를 통해 초기 KASI 데이터셋 재현 가능하게 유지
- `monthLabel`을 통한 고대 월 레이블 중의성 처리가 계속 동작하는지 검증
- 엣지 연도가 발견될 때마다 pre-1000 연도 검증 케이스 추가
- API가 안정되면 짧은 아키텍처 문서 추가 여부 결정

## 주의 항목

- `package.json` 스크립트 이름은 안정적으로 유지
- 스모크 검사는 빠르고 결정적이어야 함
- 하네스 문서는 이상적인 재작성이 아닌 실제 프로젝트 상태를 기술
- 고대 KASI 연도는 `正`과 `1`처럼 중복된 숫자 월명을 포함할 수 있음
- KASI 데이터의 초기 연도 월 스팬은 항상 `29/30`일로 제한되지 않음

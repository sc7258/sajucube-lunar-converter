# Codex 하네스

## 목적

Codex는 프로젝트가 아래 요소를 제공할 때 가장 안정적으로 동작합니다.

- 고정된 작업 범위
- 짧고 표준화된 명령 집합
- 위험한 경계 케이스를 반복 가능하게 확인하는 방법

이 프로젝트의 하네스는 의도적으로 작게 유지되며, 다음 두 가지 핵심 변경 영역에 맞춰져 있습니다.

- KST 기반 시간 해석
- 지원 범위 전체에서의 그레고리력 변환 동작

## 파일

- `AGENTS.md`: 프로젝트 전용 작업 지침
- `scripts/codex-smoke.ts`: 빠른 성공/실패 확인
- `scripts/codex-report.ts`: 중요한 경계 케이스 출력

## 표준 명령

```bash
npm run harness:smoke
npm run harness:report
npm test
```

## 권장 에이전트 흐름

1. `AGENTS.md`를 읽는다.
2. `.ai/PLAN.md`를 읽는다.
3. 가능한 한 작은 코드 변경부터 만든다.
4. 각 집중 수정 뒤에 smoke 하네스를 실행한다.
5. report 하네스로 다음 구간의 동작을 비교한다.
   - 역사 경계 전후의 그레고리력 윤년
   - 1582년 10월
   - KST 날짜 경계 근처의 UTC 시각
6. 큰 변경을 마치기 전에는 전체 Jest 스위트를 실행한다.

## 환경 메모

현재 워크스페이스는 항상 Node.js가 설치되어 있지는 않을 수 있습니다. 런타임이 없으면 하네스 파일은 최신 상태로 유지하고, Node toolchain이 준비되면 바로 실행합니다.
